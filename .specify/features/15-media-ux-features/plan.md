# Implementation Plan: Media & UX Features

**Branch**: `15-media-ux-features` | **Date**: 2026-04-01 | **Spec**: [spec.md](./spec.md)
**Input**: Feature specification covering Features 4–7

---

## Summary

Four independent UX improvements to the LoZo chat app: (4) compress images on-device before Supabase upload, (5) full-text message search using SQLite FTS5, (6) a shared media gallery screen browsable from ChatScreen, and (7) auto-save text drafts per conversation using SQLite. No backend changes required for any of these features.

## Implementation Strategy

Features are fully independent. Recommended order:

1. **Feature 7 (Drafts)** — smallest scope; pure SQLite + state changes in existing screens
2. **Feature 4 (Image Compression)** — single function change in ChatScreen + install package
3. **Feature 5 (Message Search)** — DB migration + new UI components
4. **Feature 6 (Media Gallery)** — new screen + multiple components (largest scope)

Each feature is independently testable in Expo Go before moving to the next.

## Feature 4: Image Compression — Design

### Flow
```
handleSendImage()
  → read file size via FileSystem.getInfoAsync(uri)
  → if size < 500KB OR mime === 'image/gif': skip compression
  → else: show "Compressing..." state on send button
        → manipulateAsync(uri, [{resize: {width: 1920}}], {compress: 0.8, format: SaveFormat.JPEG})
        → if compressed.size > original.size: use original
        → upload compressed (or original) via uploadToSupabase()
  → on manipulateAsync failure: upload original + show toast "Couldn't compress, uploading original"
```

### State additions to ChatScreen
```typescript
const [isCompressing, setIsCompressing] = useState(false);
```

### Key constraint
- EXIF orientation: `expo-image-manipulator` preserves orientation by default when no rotation action is applied

## Feature 5: Message Search — Design

### DB Migration
Run `initFts()` once during `initDb()` in `sqlite.ts`:
```sql
CREATE VIRTUAL TABLE IF NOT EXISTS messages_fts USING fts5(...)
CREATE TRIGGER IF NOT EXISTS messages_fts_insert ...
CREATE TRIGGER IF NOT EXISTS messages_fts_delete ...
```

### Search query
```sql
SELECT m.id, m.conversation_id, m.content, m.created_at,
       snippet(messages_fts, 2, '**', '**', '...', 32) AS highlight
FROM messages_fts
JOIN messages m ON m.id = messages_fts.message_id
WHERE messages_fts MATCH ?
ORDER BY m.created_at DESC
LIMIT 50
```

### ConversationsScreen header change
- Add search icon button to navigation header (right side, alongside or replacing gear)
- On press: Animated expansion of search bar below header (height animation from 0 to 56)
- Search bar has text input + clear/close button
- Results replace conversation list while search is active

### Scroll-to-message in ChatScreen
- Route param: `highlightMessageId?: string`
- On mount, if `highlightMessageId`: find message index in local list, `flatListRef.current?.scrollToIndex()`
- Briefly highlight with background color animation (fade in yellow → fade out, 1.5s)

## Feature 7: Message Drafts — Design

### SQLite schema addition
```sql
CREATE TABLE IF NOT EXISTS drafts (
  conversation_id TEXT PRIMARY KEY,
  text TEXT NOT NULL,
  updated_at INTEGER NOT NULL
);
```

### ChatScreen changes
```typescript
// On mount
useEffect(() => {
  const draft = getDraft(conversationId);
  if (draft) setText(draft);
}, []);

// On text change (debounced 500ms)
const draftTimer = useRef<ReturnType<typeof setTimeout>>();
const handleTextChange = (value: string) => {
  setText(value);
  clearTimeout(draftTimer.current);
  draftTimer.current = setTimeout(() => {
    if (value.trim()) saveDraft(conversationId, value);
    else clearDraft(conversationId);
  }, 500);
};

// On send
clearDraft(conversationId);

// On unmount
useEffect(() => () => clearTimeout(draftTimer.current), []);
```

### ConversationsScreen changes
```typescript
// Load all drafts on mount + on focus
const [drafts, setDrafts] = useState<Record<string, string>>({});
useFocusEffect(() => { setDrafts(getAllDrafts()); });

// In list item render
{drafts[item.conversationId] ? (
  <Text style={styles.draftPreview}>Draft: {drafts[item.conversationId]}</Text>
) : (
  <Text>{lastMessagePreview}</Text>
)}
```

Draft preview style: italic, color `#E74C3C` (red-orange)
